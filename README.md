# Simoné Rossouw | Portfolio Website

This website serves as a virtual, interactive CV.  
It is a place where all my current and future projects can be uploaded and shared with potential employers.

I used the template **Beautiful Jekyll** to create this website.

# Pages

**Home Page**  
All posts are visible to the user in a summarized format.  
For the full view of a post, the title or *Read More* can be clicked, taking the user to the complete post view.  

From the full post view, users can easily navigate back to previous posts or to any other page using the navigation bar.  

The page flow functionality was provided by Beautiful Jekyll, but the images and content are entirely my own.  

**About Me Page**  
This page highlights my general interests, current academic position, and achievements.  
It also acts as a skills page to showcase my abilities to prospective employers and teams.  

**Projects Page**  
Here users can find links to all my GitHub contributions, repositories, and forks.  

**Contact Page**  
This page provides a direct and user‑friendly way to contact me.  
While the footer includes links to my social media accounts, the contact page allows users to send me a message directly through the website.  

The functionality was provided by Beautiful Jekyll, but I created the necessary Formspree account and contact page setup to ensure messages were delivered accurately.  
Proof of successful message testing was included in the assignment submission.

# Posts

There are several ways to build a website using Beautiful Jekyll. This document focuses on the simplest method: using a fork on GitHub. For most people (including myself), this method is the recommended one.  

Even if you choose one of the [advanced installation methods](https://beautifuljekyll.com/getstarted/#install-steps-hard), I suggest reading through the easy method first. 
 

**CV Post**  
Displaying my CV as a post ensures that users can interact with it in a creative way. By sparking curiosity to explore my CV, I increase the chance of being noticed by prospective employers. This approach makes my CV memorable and helps me stand out among competitors.  

**QCAS Post**  
This post provides a visual showcase of the project. It is more engaging and user‑friendly than simply providing a repository link, as done on the Projects page.
